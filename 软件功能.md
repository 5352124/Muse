## 1. 聊天核心

聊天页面是应用的中心,围绕一条输入框和一列消息气泡展开。

### 1.1 发送消息与流式输出

- 在底部输入框打字,点击右侧发送按钮(或回车键,若开启)即可发送。
- 发送后,AI 的回复会一个字一个字地"流"出来,而非整段一次性出现。这叫"流式输出",能让你更早看到答案。
- 可以在助手设置里关闭流式输出,改为一次性返回完整回复。
- 回复过程中,顶部会出现一个可下拉的状态条,显示当前使用的模型、已用时长、本轮消耗的 Token 数。

### 1.2 多模型切换

- 聊天页顶部显示当前模型名(如 "DeepSeek-V3"、"GPT-4o")。点击可弹出模型选择列表。
- 列表按渠道(Provider)分组,切换即生效,下一条消息就用新模型回复。
- 模型旁会显示能力标签(是否支持视觉、是否支持推理、是否支持工具调用)。

### 1.3 工具调用(Function Calling)

- 当你的提问需要外部信息(如"今天天气""算一下 23×47"),AI 会自动调用对应工具,把结果拿来继续回答。
- 调用过程会在对话中显示为一张"任务卡片",里面分步列出:调用哪个工具 → 传入什么参数 → 得到什么结果 → 是否成功。
- 工具调用可以连续多轮(AI 先查天气,再查日历,再综合回答),最多 25 轮,超过会自动停止。
- 连续 3 次工具失败会提前终止,避免空转。

### 1.4 视觉与多模态

- 在输入框旁的"+"按钮里可插入图片(相册或拍照)。
- AI 能"看"图回答:描述内容、读图中的文字(OCR)、解释图表、分析截图。
- 一次可发多张图。
- 图片以 Base64 编码随消息发送(仅在模型支持视觉时生效;不支持视觉的模型只会看到文字)。

### 1.5 消息分支与重新生成

- 每条 AI 回复下方都有"重新生成"按钮。点一次就会基于上一条用户消息重新生成一个新版本。
- 多次重新生成会产生多个版本,可在消息右下角的"1/3"标签里左右切换版本。
- 也可在某条用户消息上"从此处分支",改写后续走向。
- 删除某条消息时,可选择只删这一条还是连带后续全部。

### 1.6 性能模式

- 当历史消息特别多(数百条以上),列表滚动会变卡。可在聊天设置里开启"性能模式"。
- 开启后只渲染最近若干条消息,上滑到顶时自动加载更早的内容(类似微信、QQ 的长会话体验)。
- 顶部会显示"已显示 50 / 1280 条"的指示。

### 1.7 停止与重试

- 流式回复过程中,发送按钮会变成"停止"按钮。点击立即中断。
- 中断后,已生成的部分会保留,可继续编辑或重新生成。
- 网络错误时会弹出重试按钮,点一下重新发请求。

### 1.8 深度思考(Reasoning)

- 对支持推理的模型(如 DeepSeek-R1、o1、Claude Thinking),可在输入框上方切换"思考等级":关 / 自动 / 低 / 中 / 高 / 极高。
- 思考等级越高,AI 想得越久越深,但更慢更费 Token。
- "自动"让模型自己决定;"关"则强制最小思考(若模型支持)。
- 思考过程会以可折叠的灰色区块显示在正文上方,点开可看推理细节。

### 1.9 隐私脱敏(PII)

- 在聊天设置里可开启"隐私脱敏"。
- 开启后,发送前会自动扫描消息中的手机号、身份证号、邮箱、银行卡号等敏感信息,替换为占位符(如 [PHONE])。
- AI 看不到真实号码,但回复中如有占位符会还原成原值,保证你阅读时正常。

### 1.10 Token 统计

- 每条 AI 回复下方显示"输入 / 输出 / 合计"Token 数。
- 顶部状态条显示本轮用时、速度(tokens/秒)。
- 历史总消耗在"设置 → 关于"里可查。

### 1.11 会话管理

- 首页侧边栏列出全部会话,可置顶、重命名、归档、删除。
- 新建会话可基于某个助手模板。
- 切换会话即切换上下文,各会话记忆独立。
- 长按会话可多选批量操作。

### 1.12 复制、分享、导出

- 长按单条消息可复制纯文本、复制 Markdown、转发到其他会话、分享到系统分享菜单。
- 整段会话可导出为 Markdown 文件、纯文本、JSON。
- 支持截图分享(把整段对话渲染成一张长图,带气泡样式)。

### 1.13 输入栏(InputBar)

- 支持多行输入,自适应高度(最高 6 行后内部滚动)。
- 左侧"+"按钮:插入图片、文件、快捷消息、记忆、知识库引用。
- 右侧发送按钮,流式中变为停止按钮。
- 支持草稿(切换会话时保留未发送的输入)。
- 快捷消息:可预设常用句子,一键插入。

## 2. 助手管理

"助手"是一套预设的角色配置,决定 AI 的人设、语气、可用工具、可用知识、记忆策略等。

### 2.1 预设助手

应用内置 8 个开箱即用的角色:
- **通用助手**:无特定人设,适合日常问答。
- **写作助手**:擅长文章、文案、润色。
- **翻译官**:多语言互译,保留语气。
- **代码助手**:编程问题、代码生成、调试。
- **学习导师**:讲解知识、出题、批改。
- **头脑风暴**:发散创意、列举方案。
- **角色扮演**:按设定的人物对话。
- **效率秘书**:日程、待办、提醒。

每个预设助手都预先配置了系统提示词、推荐模型、默认工具集。

### 2.2 自定义助手

- 在助手页点"+"新建,可从零开始或复制现有助手为模板。
- 可填写的字段超过 35 项,分 5 大类(详见 2.4)。
- 支持头像:可上传图片、选 emoji、或用名字首字母。
- 支持标签分组(如"工作""娱乐"),便于筛选。

### 2.3 助手字段一览

每个助手可配置:

**基础信息**:名字、头像、专属模型、温度、Top-P、最大 Token、上下文消息数、推理等级、是否流式输出。

**提示词**:系统提示词(角色与规则)、消息模板(支持 {{date}}、{{time}}、{{session_id}} 等变量)、预设消息(JSON 数组,可塞入示例对话)。

**扩展能力**:快捷消息、Lorebook(世界书)、提示词注入、Skill、MCP 服务、工具。

**记忆**:是否启用记忆、用全局记忆池还是独立池、是否注入最近会话摘要、是否注入当前时间、手动记忆条目。

**高级**:背景图 URL、背景透明度、渐变遮罩、自定义请求头(JSON)、自定义请求体(JSON)、标签。

### 2.4 助手设置页(AgentSettingsPage)

设置页分 5 个分区:
1. **基础**:模型与采样参数。
2. **提示词**:人设与模板。
3. **扩展**:挂载的各种资源。
4. **记忆**:记忆策略与手动记忆。
5. **高级**:外观与请求定制。

### 2.5 助手导入导出

- 可把助手配置导出为 JSON 文件,方便分享或备份。
- 支持导入本应用格式的 JSON。
- 额外支持导入 **SillyTavern** 角色卡(PNG/CardJSON 两种格式),自动映射字段。

### 2.6 聊天页快速切换助手

- 聊天页顶部显示当前助手名,点击弹出助手选择列表。
- 切换后下一条消息即用新助手,历史消息保留。
- 也可"在新会话中打开"该助手。

### 2.7 资源关联

每个助手可独立挂载 7 类资源:
- 工具(本地工具)
- Skill(技能)
- MCP 服务
- 知识库
- Lorebook(世界书)
- 快捷消息
- 提示词注入片段

### 2.8 Regex 替换

- 可为助手配置正则替换规则,作用在 AI 回复上。
- 例如:把回复中所有 `[微笑]` 替换成 emoji;或删除思考标签 `<think>...</think>`。
- 支持多条规则按顺序执行。

### 2.9 World Book(世界书)

- 类似 SillyTavern 的 Lorebook,用于补充世界设定。
- 每条目有触发关键词,当对话中出现关键词时,对应条目被注入到提示词。
- 适合角色扮演、长篇创作时维护人物、地点、事件设定。

### 2.10 Mood(心情标签)

- 助手可配置心情标签,影响回复语气。
- 心情可手动切换,也可由 AI 根据上下文自动判定。

## 3. 模型与渠道(Provider)

"渠道"(Provider)是 AI 服务的入口配置,包含地址、密钥、可用模型列表。

### 3.1 预置渠道

应用内置 27 个常用渠道的开箱即用配置,只需填入自己的 API Key 即可使用:
- OpenAI、Anthropic、Google Gemini、DeepSeek、智谱、月之暗面、百川、零一万物、阿里通义、字节豆包、阶跃星辰、MiniMax、SiliconFlow、OpenRouter、Together、Groq、Fireworks、Mistral、Cohere、Perplexity、Grok、OpenAI 兼容(Console Go / new-api / one-api 等中转站)、本地 Ollama、本地 LM Studio、本地 KoboldCpp、本地 llama.cpp、本地 Jan。

### 3.2 渠道配置

- 每个渠道可填:名称、Base URL、API Key、是否启用、自定义请求头、自定义请求体、能力矩阵(是否支持工具、视觉、推理等)。
- 支持多 Key 轮换:同一渠道可填多个 Key,请求时自动轮换或随机选择,某个 Key 失败自动切换下一个。

### 3.3 模型配置

- 每个渠道下可配置多个模型,每个模型填:id、显示名、上下文长度、是否支持视觉/推理/工具、价格(可选)。
- 支持从渠道 API 拉取模型列表(/v1/models 接口),自动填充,免去手输。
- 模型 id 可包含 `provider/` 前缀(OpenRouter 等中转站需要),应用默认原样透传,不擅自剥离。

### 3.4 SiliconFlow 一键填入

- SiliconFlow 渠道支持"一键填入":只需粘贴 API Key,模型列表、能力矩阵自动补全。

### 3.5 余额查询

- 部分渠道(OpenAI、SiliconFlow、DeepSeek、智谱等)支持余额查询。
- 在渠道详情页点"查询余额",显示剩余额度与到期时间。

### 3.6 多 Key 轮换

- 同一渠道可挂多个 Key。
- 调用策略可配:顺序轮换、随机选择、失败重试下一个。
- Key 失效(401/429)会自动标记并切到下一个,不打扰用户。

### 3.7 聊天页快速切换

- 聊天页顶部的模型选择器,跨渠道列出全部已启用模型,一键切换。

### 3.8 模型能力推断

- 应用内置一张"模型能力表",根据模型 id 自动推断是否支持视觉、推理、工具调用、最大上下文等。
- 例如 `gpt-4o` 推断为支持视觉 + 工具 + 推理(minimal);`deepseek-reasoner` 推断为支持推理但不支持工具。
- 用户也可手动覆盖推断结果。

### 3.9 OAuth 登录

- 部分渠道(如 Google Gemini)支持 OAuth 授权登录,无需手动复制 API Key。
- 在渠道详情页点"登录",跳转浏览器完成授权后自动回填 Token。

### 3.10 全局聊天偏好

在"设置 → 聊天"里可配:
- 默认模型(新会话用哪个)。
- 默认助手。
- 性能模式开关与阈值。
- 是否发送思考过程。
- 是否显示 Token 统计。
- 输入栏行为(回车发送 vs 换行)。

## 4. 长期记忆系统

记忆让 AI 能"记住"跨会话的事实与经历,下次对话时自动参考。

### 4.1 四层记忆架构

记忆按抽象程度分四层:
1. **事实层(Fact)**:原始事实,如"用户是程序员""用户喜欢咖啡"。每条带重要程度(0/1/2 星)。
2. **会话摘要层(Summary)**:每次会话结束后自动生成的摘要。
3. **编译层(Compile)**:周期性把事实 + 摘要"编译"成更高层的认知,如"用户偏好简洁直接的回复风格"。
4. **深层(Deep)**:处理层,无独立存储,用于在线推理时综合前三层给出"AI 对你的理解"。

### 4.2 作用域

- 每条记忆带"作用域"标签:全局、某助手、某会话。
- 助手可配置"用全局记忆池"还是"独立记忆池"(实验功能)。
- 切换作用域即可查看不同范围的记忆。

### 4.3 重要程度

- 事实层每条记忆有 0/1/2 星三级重要程度。
- 2 星=关键(用户身份、核心偏好);1 星=重要;0 星=一般。
- 重要程度影响注入优先级(上下文紧张时优先保留高星)。

### 4.4 筛选

记忆页支持四维筛选:
- **作用域**:全局 / 助手 / 会话。
- **重要程度**:全部 / 0 星 / 1 星 / 2 星。
- **时间**:全部 / 今天 / 本周 / 本月。
- **类型**:全部 / 事实 / 摘要 / 编译 / 里程碑。

### 4.5 列表视图与时间轴视图

- **列表视图**:按四层折叠展开,每层独立增删改。
- **时间轴视图**:按月份倒序排列,带圆点节点和连接线,重要事件标星,视觉上看像一条"记忆长河"。
- 顶部一键切换两种视图。

### 4.6 手动增删改查

- 任何一条记忆都可手动编辑内容、改重要程度、改作用域、加标签。
- 删除有二次确认。
- 可手动新增"元事实",直接告诉 AI"我是 XX""我喜欢 XX"。

### 4.7 经验库

- 独立于四层记忆的"最佳实践库"。
- 每条经验有:标题、详情、分类、标签。
- AI 在类似任务中会自动参考。
- 适合记录"如何调试某类 bug""如何写某类文档"等可复用 know-how。

### 4.8 自动编译

- 后台定时把事实 + 摘要编译成更高层认知。
- 也可在记忆页点"立即编译"手动触发。
- 编译过程有步骤详情,失败会重试。

### 4.9 记忆注入到提示词

- 每次发消息时,系统会自动把相关记忆拼接到提示词里。
- 注入策略:按作用域 + 重要程度 + 时间衰减打分,取 Top N 条。
- 助手可关闭记忆或改为独立池。

### 4.10 导入导出

- 整个记忆库可导出为 JSON 备份。
- 支持从 JSON 导入(合并到现有库)。

### 4.11 全文搜索

- 记忆页顶部搜索框,支持全文检索(基于 FTS4)。
- 输入关键词即列出所有命中条目,高亮匹配词。
- 搜索带 300ms 防抖,避免每次按键都查库。

### 4.12 向量化与重排

- 记忆条目会用本地 ONNX 模型转向量(无需联网)。
- 检索时按向量相似度排序,再用 ONNX 重排模型精排,提升相关性。
- 这是"语义检索",能理解同义词(如搜"工作"也能命中"上班")。

## 5. 知识库与智能检索(RAG)

知识库让你把文档塞给 AI,问问题时 AI 会先从文档里找答案再回复,避免胡说。

### 5.1 知识库管理

- 可创建多个知识库,每个独立管理文档。
- 知识库可绑定到某个助手(助手级 RAG)或全局共享。
- 支持重命名、删除、导入导出。

### 5.2 文档导入

支持的格式:
- **PDF**(.pdf):含扫描件(走 OCR)。
- **Word**(.docx)。
- **PowerPoint**(.pptx)。
- **Excel**(.xlsx)。
- **电子书**(.epub)。
- **纯文本**(.txt / .md)。
- **图片**(.jpg / .png):走 OCR 提取文字。

可批量导入,导入后显示文档列表与处理状态。

### 5.3 OCR 文字识别

- 扫描版 PDF 和图片会自动走 OCR。
- OCR 在本地完成,不上传图片到第三方。

### 5.4 文档分块

文档太长无法整段塞给 AI,需要切块:
- **按字符**:固定字数一块。
- **按 Token**:按模型 Token 数切块,更精准。
- **Markdown 感知**:按标题层级切,保留结构。
- 每块默认带相邻重叠(overlap),避免切断语义。

### 5.5 向量化(Embedding)

把文本块转成向量,用于相似度检索:
- **本地 ONNX**:设备端运行,不联网,隐私好。
- **云端 API**:OpenAI / SiliconFlow / 智谱等 Embedding 服务,速度快质量高。
- **关键词**:不做向量化,纯关键词匹配(BM25),适合精确术语检索。

### 5.6 向量存储

- 向量存在本地 SQLite,用余弦相似度检索。
- 无需额外数据库服务。

### 5.7 检索策略

提问时从知识库找相关块:
- **Top-K**:取相似度最高的 K 条(默认 5)。
- **重排(Rerank)**:用交叉编码器精排,提升 Top 准确率。
- **RRF 融合**:向量检索 + 关键词检索结果融合,兼顾语义和精确匹配。

### 5.8 重排模型(Rerank)

- 本地 ONNX 交叉编码器,对候选块重新打分排序。
- 不联网,隐私安全。

### 5.9 引用标注(Citation)

- AI 回复中会标注引用来源,如"[1]"指向某个文档的某段。
- 点击引用可跳转到原文位置,方便核对。
- 这让 AI 回答可溯源,而非凭空生成。

### 5.10 MCP 服务接入

- 知识库可作为 MCP 服务暴露给其他应用调用。
- 也可反向:接入外部 MCP 知识源作为检索来源。

### 5.11 助手级 RAG 覆盖

- 每个助手可独立挂载知识库,互不干扰。
- 助手 A 挂"产品文档库",助手 B 挂"法律条文库"。

### 5.12 RAG 设置页

可调参数:
- 分块大小、重叠。
- Top-K 数量。
- 是否启用重排。
- Embedding 模型选择。
- 检索阈值(相似度低于多少丢弃)。

## 6. 工具系统

工具让 AI 能执行真实动作:查天气、读剪贴板、发短信、生成图片等。

### 6.1 工具编排器(ToolOrchestrator)

- 负责接收 AI 的工具调用请求 → 执行 → 把结果回传给 AI。
- 支持并行执行(同一轮多个工具同时跑)。
- 单工具超时 2 分钟。
- 失败自动重试,连续 3 次失败终止本轮。

### 6.2 内置工具分类

应用预装约 70 个工具,按类别:

**Web 工具**:web_search(网页搜索)、web_fetch(抓取网页内容)、http_get、http_post。

**文件工具**:read_file、write_file、list_installed_apps。

**系统工具**:get_current_time、calculator、echo、clipboard_read、clipboard_write、url_encode、url_decode、base64_encode、base64_decode、hash_text、generate_uuid、random_number、generate_password、json_pretty、speak_text(TTS)。

**设备工具**:get_weather、get_location、get_battery_info、get_device_info、get_storage_info、get_memory_info、get_display_info、get_cpu_info、get_sensors_list、get_brightness、set_brightness、get_volume、set_volume、toggle_flashlight、vibrate、get_foreground_app、get_wifi_info、get_bluetooth_devices、make_phone_call、open_maps、get_network_info、ping_host、dns_lookup、get_public_ip。

**通讯录与短信**:get_contacts_count、get_contacts_list、add_contact、send_sms、send_email。

**日历与提醒**:calendar_today、add_calendar_event、set_alarm、set_timer、schedule_reminder、cancel_reminder、list_reminders。

**应用与系统设置**:open_app、open_url、open_system_setting、toggle_wifi、toggle_bluetooth、screen_time、get_recent_notifications。

**资源库与速记**:resource_add、resource_list、resource_search、resource_get、resource_delete;quick_note_add、quick_note_list、quick_note_search、quick_note_get、quick_note_update、quick_note_delete、quick_note_pin。

### 6.3 媒体生成工具

- **generate_image**:根据文字描述生成图片,支持指定尺寸、质量、风格、数量。消耗绘图 API 额度。
- **generate_video**:根据描述生成短视频(5/10 秒,720p/1080p),消耗 Kling 视频生成额度。
- **generate_qr_code**:把任意文本(链接、WiFi、联系方式)转成二维码图片。

### 6.4 浏览器自动化

基于无头 WebView 实现:
- browser_navigate:打开网页。
- browser_click:点击元素。
- browser_type:在输入框输入文字。
- browser_extract:提取页面内容。
- browser_scroll_bottom:滚动到底。
- browser_get_html:获取页面 HTML。

适合"帮我打开某网站,登录后抓取订单列表"这类复合任务。

### 6.5 工作区文件管理

- workspace_list / read / write / delete / mkdir / move。
- 在应用沙盒内管理文件,AI 可读写但隔离于系统其他目录。

### 6.6 JavaScript 沙盒

- execute_javascript:在隔离的 WebView 里运行 JS 代码。
- 适合计算、数据处理、原型验证。

### 6.7 MCP 桥接(MCP Bridge)

- 支持 Model Context Protocol,可接入第三方 MCP 服务。
- MCP 服务的工具会自动注册到工具列表,名字带 `mcp_服务器名_工具名` 前缀,避免与内置工具冲突。
- 一个应用可同时接入多个 MCP 服务。

### 6.8 插件系统

内置 5 个插件:
- image-gen:图片生成增强。
- beautify:文本美化。
- media:媒体处理。
- mcp-bridge:MCP 桥接。
- office:Office 文档处理。

插件有清单(manifest),声明权限(capabilities)和激活事件。高风险权限(如 full-access)会高亮提示。

### 6.9 工具审批

- 高风险工具(如发短信、拨电话、删文件)首次调用时弹窗让你确认。
- 可在设置里选择"总是询问""每次询问""从不询问(自动允许)"。
- 审批记录可在工具历史里查看。

### 6.10 工具历史

- 每次工具调用都记录:时间、工具名、参数、结果、耗时、是否成功。
- 在聊天页对应的消息卡片里可展开查看。
- 也可在设置 → 工具历史里全局查看。

### 6.11 Skill 技能系统

- Skill 是"可分发的工具",比内置工具更灵活。
- 内置约 30 个 Skill:生成图片、翻译、生成二维码、表情包相关(list_stickers / send_sticker)、记忆操作(pin_memory / recall_experience / record_experience)、待办(todo_write)、卡片(show_card)、通知(notify)、委派子任务(subagent_task)等。
- 支持自定义 Skill:用 JSON 描述参数,实现可挂 Kotlin 代码或调用其他工具。
- 支持从文件导入 .skill.json。
- 导入时做安全审查:防 prompt injection、防伪装系统 id、防过宽权限。

### 6.12 委派代理(delegate_agent)

- AI 可创建子代理处理子任务,主代理继续主线。
- 适合"同时调研三个方向再汇总"这类并行任务。
- 可手动取消委派。

### 6.13 表情包库

- 内置表情包,sticker 工具让 AI 在合适时机发表情。
- 可在设置里开关,并控制发送概率(0-100%)。
- 概率命中时才向 AI 暴露 sticker 工具,实现"偶尔发表情"的自然效果。

### 6.14 工具页(ToolsScreen)

- 集中管理所有工具:内置工具、Skill、MCP 工具、插件。
- 每个工具显示:名称、描述、参数、风险等级、所属类别。
- 可启用 / 禁用单个工具。
- 末尾有"内置插件"分组,展示 5 个插件的清单信息。

## 7. 外观与个性化

### 7.1 预设主题

内置 12 套主题:
- 暖纸(warm-paper,默认)
- 鼠尾草绿(SageCalm)
- 珊瑚粉(CoralWhisper)
- 薰衣草梦(LavenderDream)
- 琥珀暖(AmberWarmth)
- 以及深色模式对应的 6 套暗色主题。

每套主题定义了主色、背景、卡片色、文字色等完整色板。

### 7.2 动态取色

- 可开启"跟随壁纸动态取色",主色从当前壁纸提取,自动协调。
- 切换壁纸即自动更新主题色。

### 7.3 自定义主题

- 在预设基础上自由调整:主色、次色、背景、卡片、文字、边框等。
- 自定义主题可保存命名,方便切换。

### 7.4 字体

- 字号:小 / 中 / 大 / 超大。
- 字体家族:默认 / 等宽 / 系统三种。
- 等宽字体适合代码场景。

### 7.5 圆角与形状(MuseShapes)

- 全局圆角风格:小 / 中 / 大。
- 所有卡片、按钮、输入框统一应用,视觉一致。
- 采用 iOS 风格圆角,而非 Android 原生方角。

### 7.6 设置页结构

设置主页分组:
- 外观与个性化(主题 / 字体 / 用户资料)
- 聊天(消息显示 / 交互 / 高级)
- 隐私与安全
- 备份与恢复
- 关于

iOS 风格分组卡片,圆角带分隔线。

### 7.7 聊天设置

可调:
- 字号、行距。
- 气泡样式(圆角大小、对齐方式)。
- 是否显示头像、时间戳、Token 统计。
- 性能模式开关与阈值。
- 输入栏行为(回车发送 vs 换行、草稿保留)。
- 思考过程显示与折叠默认状态。

### 7.8 PII 隐私脱敏

- 见 1.9。在聊天设置里开关。
- 可自定义脱敏规则(哪些模式算手机号、邮箱等)。

### 7.9 审计日志

- 记录敏感操作:删除会话、导出数据、修改密钥、工具审批通过等。
- 在设置 → 审计日志里查看,可按时间 / 类型筛选。

### 7.10 备份与恢复

支持三种备份目标:
- **本地**:导出到设备存储或下载目录。
- **WebDAV**:自建 NAS 或网盘的 WebDAV 服务。
- **S3**:兼容 S3 协议的对象存储(MinIO、阿里云 OSS、AWS S3 等)。

可配自动备份频率(每日 / 每周 / 手动)。备份内容含:会话、助手、记忆、知识库索引、设置(不含密钥)。

### 7.11 生物识别解锁

- 可开启指纹 / 面容解锁应用。
- 启动时需生物识别通过才能进入。
- 失败若干次后回退到 PIN 码。

### 7.12 应用锁

- 可设 PIN 码锁,启动或回到前台时要求输入。
- 可设自动锁定延时(立即 / 1 分钟 / 5 分钟 / 30 分钟)。

### 7.13 多语言

支持 6 种语言:
- 简体中文(默认)
- 繁体中文
- English
- 日本語
- 한국어
- Русский

跟随系统语言自动切换,也可手动指定。

### 7.14 Debug 调试页

- 实时日志查看(支持关键字过滤、级别筛选)。
- 关键字高亮(黄色背景)。
- 一键复制当前过滤后的日志到剪贴板。
- 导出完整日志到文件。
- 查看应用版本、设备信息、内存占用。

### 7.15 启动页(SplashScreen)

- 应用启动时短暂显示品牌标识。
- 渐隐过渡到主界面。

### 7.16 关于页

- 版本号、构建号、更新日志。
- 检查更新(若发布渠道支持)。
- 开源许可列表。
- 反馈入口。

### 7.17 引导页(Onboarding)

- 首次启动展示功能引导:聊天、助手、记忆、知识库、工具五大能力各一页。
- 可跳过,后续在设置里可重新查看。

## 8. 辅助与扩展能力

### 8.1 通知监听

- 可授权应用读取系统通知。
- AI 能看到通知内容,回答"刚才谁给我发了什么消息"。
- 可设白名单,只监听指定应用的通知。
- 通知可触发自动回复或提醒。

### 8.2 截图分享

- 把任意一段对话渲染成图片长截图,带气泡样式。
- 可直接分享到其他应用。
- 适合保存精彩对话或发社交媒体。

### 8.3 定时任务

- 可设定时任务:在指定时间自动执行某操作。
- 例如"每天早上 8 点用某助手发今日天气和待办"。
- 任务可启用 / 暂停 / 删除。
- 支持一次性与重复(每日 / 每周 / 自定义)。

### 8.4 WebDAV 同步

- 见 7.10。也可单独用于同步助手配置或记忆库。

### 8.5 S3 同步

- 见 7.10。适合企业内部部署。

### 8.6 群聊

- 多个助手在同一会话里轮流或同时回复。
- 可模拟"圆桌讨论":写作助手 + 翻译官 + 代码助手共同回答。
- 可设发言顺序或自由发言。

### 8.7 收藏

- 任意消息可收藏,在"收藏夹"里集中查看。
- 支持分类与搜索。
- 适合保存金句、常用代码片段、重要结论。

### 8.8 报告生成

- 可让 AI 把一段会话总结成报告(会议纪要、学习笔记、调研报告等)。
- 报告支持导出为 Markdown / TXT。
- 可自定义报告模板。

### 8.9 桌面小部件(Widget)

- 桌面添加小部件,快速发起对话或查看最近会话。
- 多种尺寸(2×2 / 4×1 等)。
- 点击即跳转到对应会话。

### 8.10 快捷方式

- 长按应用图标弹出快捷菜单:新建会话、打开某助手、打开最近会话。
- 也可为单个助手创建桌面快捷方式,点击直接进入该助手会话。

### 8.11 分享接收

- 在系统分享菜单里选 Muse,可把文字 / 图片 / 文件直接发到某会话。
- 自动新建会话或追加到指定会话。
- 适合"看到一篇文章,丢给 AI 总结"。

### 8.12 Deep Link

- 应用支持 `muse://` 协议,可从网页或其他应用跳转到指定页面。
- 例如 `muse://chat/assistant_id=xxx` 直接打开某助手会话。

### 8.13 开机自启

- 可在设置里开启"开机自启"。
- 开机后自动启动应用(部分品牌手机需在系统设置额外授予自启动权限)。
- 配合定时任务可实现"开机后立即执行晨间例行"。

### 8.14 前台服务

- 长时间运行的任务(流式回复、定时任务、MCP 服务)在前台服务中运行。
- 状态栏显示持久通知,避免被系统杀进程。
- 通知可隐藏或最小化。

### 8.15 Ktor Web 服务器

- 应用内置 Ktor Web 服务器,可作为 MCP 服务端或本地 API 暴露给其他工具调用。
- 端口可配。
- 适合把 Muse 当作本地 AI 网关使用。

### 8.16 二维码

- 可把任意内容(链接、文本、配置)生成二维码分享。
- 也可扫描二维码导入助手配置或会话。

### 8.17 会话生成管理器

- 后台统一管理多个并发的会话生成任务。
- 防止重复触发、资源冲突。
- 支持任务排队与取消。
